Create sf_client.py at project root
sf_client.py
import os
from syncforge import SyncForge

# Initialize client for {{ project.name }}
sf = SyncForge(
    api_key=os.environ.get('SYNCFORGE_API_KEY', '{{ api_keys.first.key|default:"sf_live_YOUR_KEY" }}')
)

# Register tables programmatically (runs once)
sf.create_table('items', sync_mode='event')
Zero-Code Syncs 🚀 Add the @sync_model decorator to your models. SyncForge will automatically trigger on every Create, Update, and Delete!
3
Zero-Code Syncs 🚀
models.py
from django.db import models
from sf_client import sf
from syncforge.django import sync_model

# Auto-registers table & syncs on all DB changes ⚡
@sync_model(sf)
class Item(models.Model):
    name = models.CharField(max_length=100)
    price = models.IntegerField()
How it works: The sf.refresh('table') command sends a lightweight signal to our servers. SyncForge handles cache invalidation and distributes the fresh data delta to all connected client devices automatically. Your database only receives 1 query, not 10,000.
