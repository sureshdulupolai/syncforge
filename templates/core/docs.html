Introduction to SyncForge

SyncForge empowers you to control exactly when data is synchronized between your database and your client applications.

Triggering a Manual Refresh (Django Example)

When you're using "Manual Mode" or "Hybrid Mode", you need to inform SyncForge when your database changes. You can easily integrate this into your existing Django application using our official Python SDK.

# views.py

from django.shortcuts import render
from .models import Product
import syncforge

def create_product(request):
    # 1. Update your local database
    Product.objects.create(name='New Premium Item', price=99.99)

    # 2. Tell SyncForge that the 'products' table has changed
    client = syncforge.Client(api_key='sf_live_YOUR_API_KEY')
    client.refresh('products')

    return render(request, 'success.html')

How it works

The moment you call client.refresh('products'), SyncForge immediately invalidates its internal cache for that table. The next time a client device requests the products data, SyncForge will fetch the fresh Delta from your server, cache it, and distribute it globally.
