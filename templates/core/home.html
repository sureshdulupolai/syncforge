Architecture

The Hybrid Cache Engine

SyncForge combines ultra-fast RAM caching with an encrypted local disk fallback to minimize database queries while keeping data consistent.

Read Flow (Data Retrieval)

1. Check Memory Cache (RAM)

SyncForge checks if the requested data exists in fast server memory.

RAM Hit: 0.1ms latency

2. Check Encrypted Disk Cache

If memory is empty, SyncForge reads the encrypted cache file from local disk storage.

Disk Hit: 2.0ms latency

3. Database Fetch & Save

If both caches miss, it queries PostgreSQL/MySQL and automatically updates the cache for future reads.

DB Hit: 50.0ms latency

Write Flow (Invalidation)

1. Write to Database

Your application updates or inserts a record into the primary database.

2. Invalidation Hook

An ORM hook or explicit SDK call triggers `sf.refresh('table_name')`.

3. Evict Local Cache

The local server's memory is instantly wiped for the stale data.

4. Broadcast to Cluster

A light refresh signal is sent asynchronously to all other active application nodes.

Platform Features

Engineered for Production

Intelligent Heat Engine

SyncForge actively tracks payload access frequencies. Hot objects stay in memory, while cold data gracefully flushes, completely preventing OOM crashes.

Object Deduplication

Multiple cache payloads containing identical underlying data automatically map to shared memory references, radically reducing redundant RAM overhead.

Smart Serialization

Bypasses slow static pickling logic. The SDK dynamically detects primitive dicts and falls back to structural `orjson` serialization for lightning-fast reads.

Request Coalescing

Stops backend Notification Storms. Identical concurrent validation triggers automatically merge into a single network API call to preserve server integrity.

Stale-While-Revalidate

Eliminates cache stampedes. When data expires, the first thread locks the fetch while all subsequent threads instantaneously read the stale payload without blocking.

Piggybacked Telemetry

No extra network logs needed. SyncForge injects runtime analytics seamlessly into existing background HTTP headers to track bandwidth without I/O penalties.
