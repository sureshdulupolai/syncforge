The "0 DB Calls" Advantage

SyncForge is engineered to give your application massive speed by completely eliminating redundant database queries.

Where & When to use it?

Best for Static Data: E-Commerce Products, Blog Articles, and Global Configurations.

What happens? If you set your database cache to a 100-day timer, the very first user loads the data. For the next 100 days, every subsequent user instantly receives the data with 0 Database Calls. Server load drops to practically zero, and UI speed multiplies instantly.

When NOT to use it?

Avoid Highly Dynamic Data: Live Chats, Stock Market Tickers, and Real-Time Comments.

The Warning: If data changes every second, using SyncForge will force you to call the refresh() function continuously. This will increase database load rather than decrease it. Use standard WebSockets for those specific tables instead.

How to Use SyncForge

Integrate our framework into your application in less than 5 minutes. No complex infrastructure changes required.

1

Install the Framework

Add SyncForge to your project. Our SDK acts as a lightweight wrapper around your existing database calls, capturing mutations without adding overhead.

# Install via pip — just like FastAPI
pip install syncforge

# Or for Node.js
npm install syncforge
2

Initialize with your API Key

Create a free Developer Account, generate an API key in the Dashboard, and instantiate the client in your backend.

import os
from syncforge import SyncForge

# Dead simple — like Flask or FastAPI
sf = SyncForge(
    api_key=os.environ.get('SYNCFORGE_API_KEY')
)
3

Configure Table Modes

Go to the SyncForge Dashboard. For each table, decide the behavior: Manual for static tables, Schedule for semi-static, or Event for high-velocity data.

4

Trigger Manual Updates

If you selected Manual Mode, automatic polling is turned off. Your clients will never hit your database until you explicitly call the `refresh` method.
