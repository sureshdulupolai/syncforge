Dashboard Feature

SyncForge CMD

The SyncForge CMD is a powerful, AWS-style terminal built directly into your Project Dashboard. It allows you to monitor backend events and inspect table caching status in real-time, all without ever exposing your actual database row data.

Overview

Every SyncForge project comes with a dedicated CMD tab. Simply open your project in the dashboard and click the CMD tab. The terminal provides a secure window into the metadata that powers your synchronised clients.

Available Commands

logs

Streams the chronological audit log of your project's events. This command lets you track exactly when tables are created, when API keys are generated, and most importantly, when cache refresh signals are triggered by your backend. 

$ logs
--- SyncForge Event Logs ---
[2026-06-25 14:00:00] [Table Created] Registered table ID: users
[2026-06-25 14:05:30] [Sync Refresh] Manually refreshed table 'users'. New version: 2
table <table_name>

Fetches the real-time metadata profile for a specific table.

Security Note: SyncForge utilizes a Zero-Knowledge architecture. Your actual database row data is securely fetched and stored exclusively on your client environments (inside their local encrypted caches). Therefore, this command returns the performance metrics and structural profile of the cache, not the data payload itself. 

$ table users
--- Fetching Data Profile for 'users' ---
Note: Actual row data is securely cached in your local environment.

Database Status: Active
Current Version: v2
Sync Mode:       Event-driven caching
Storage Mode:    RAM & Disk Storage
Encryption:      Enabled (AES-256)
Cache Hits:      1042
clear

Wipes the terminal screen clean, giving you a fresh workspace.

Local Terminal Commands (SDK)

While the SyncForge CMD monitors your live metadata remotely, some commands (like speed benchmarking) require access to your physical database and are run directly in your local code/terminal.

sf.speed(queryset, registered_table, disk)

Runs a real-time, zero-knowledge speed benchmark comparing your Database fetch time vs the SyncForge Cache fetch time. Prints the results directly to your local server terminal. 

# Local Python Code
sf.speed(User.objects.all())
$ python manage.py runserver

🚀 --- SyncForge Speed Benchmark ---
Table Name:    __sf_internal_speed_test__
Original DB:   245.50 ms
SF RAM Cache:  2.10 ms
Performance:   116.9x Faster
