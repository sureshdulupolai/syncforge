Django Local (Offline) Mode

Develop your Django application offline without ever hitting SyncForge cloud servers. In dev_mode=True, SyncForge mocks every single network call and prints the A-Z request and response lifecycle directly to your local terminal.

1. Enabling Dev Mode

To start mocking, simply set dev_mode=True. You do not need to provide an API key. 

from syncforge import SyncForge

# api_key is ignored in dev_mode
sf = SyncForge(dev_mode=True, backend='in_memory')

2. A-Z Event Mocks & Terminal Logs

Because SyncForge intercepts every Django hook (like @sync_model), it will generate mocked JSON payloads in your console. Here is the comprehensive breakdown of every feature's request and response log.

Feature: Model Save Intercept (Refresh)

sf.refresh()

Triggered automatically when a Django model decorated with @sync_model is saved or deleted.

🌐 [SyncForge Local Dev] Simulated Network Request
► METHOD: POST
► URL:    https://syncforge.dev/api/v1/sync/core_product/
◄ RESPONSE: {
  "success": true,
  "status": "ok",
  "table": "core_product",
  "project": "local_mock_project",
  "message": "[Dev Mode] Sync Triggered",
  "data": {
    "action": "refresh",
    "table": "core_product",
    "django_signal": "post_save"
  }
}

Feature: Fetch Table Metadata

sf.get_table()

When Django requests the latest cache version for a specific table.

🌐 [SyncForge Local Dev] Simulated Network Request
► METHOD: GET
► URL:    https://syncforge.dev/api/v1/sync/auth_user/
◄ RESPONSE: {
  "success": true,
  "status": "ok",
  "table": "auth_user",
  "message": "[Dev Mode] Table Fetched",
  "data": {
    "version": 1,
    "last_sync": "2026-06-27T12:00:00Z"
  }
}

Switching back to Live (Cloud) Mode

When you are ready to deploy or want to see live analytics in the SyncForge Dashboard, you must switch back to Live Mode.

  1. Remove dev_mode=True from your initialization.
  2. Provide your valid SYNCFORGE_API_KEY (starting with sf_live_).
  3. Once switched, terminal prints will stop, and all logs will instantly begin streaming to your Remote Dashboard CMD.
# Back to Live Cloud Mode
sf = SyncForge(
    api_key=os.environ.get('SYNCFORGE_API_KEY'),
    backend='redis'
)
