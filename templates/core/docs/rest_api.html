API Reference

REST API

If you're not using Python, you can integrate SyncForge using raw HTTP requests. All endpoints require the X-API-Key header.

Base URL

https://syncforge.dev/api/v1
POST /sync/{table_name}/

Signal that a table's data has changed. Increments the version number and logs a SyncEvent.

Headers

X-API-Key Required Your project API key (sf_live_...)
X-SF-Timestamp Optional Unix timestamp for replay protection
X-SF-Signature Optional HMAC-SHA256 signature of the request

Response (200 OK)

{ "status": "ok", "message": "Sync triggered for table `users` in project `MyApp`.", "table": "users", "sync_mode": "Event — On INSERT / UPDATE / DELETE", "database_calls_saved": 1042 }

2. Reading Data (Cache-Aside Pattern)

When building your own client without the Python SDK, you should implement the cache-aside pattern locally. Follow these 3 steps:

  1. 1
    Get Local Cache: Instantly fetch pre-warmed data from your local memory. Zero database hits.
  2. 2
    Query DB & Save: If memory is empty (cache miss), execute the DB query and automatically save the result back to memory.
  3. 3
    Combine Both: Use a simple if/else block to ensure only the very first user triggers a database query.

Pseudo-code Implementation

// Pseudo-code for caching logic
if (local_cache.get("product")) {
    // 1. Cache Hit: Data found in memory! DB query skipped entirely.
    log = local_cache.get("product");
} else {
    // 2. Cache Miss: Execute DB query and automatically save to memory
    log = database.query("SELECT * FROM products");
    local_cache.set("product", log);
}
// 3. Return data exactly as it came from the DB
return render("home", {"product": log});
