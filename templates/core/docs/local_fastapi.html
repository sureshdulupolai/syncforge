FastAPI Local (Offline) Mode

Develop your FastAPI application offline without ever hitting SyncForge cloud servers. In dev_mode=True, SyncForge mocks every single network call and prints the A-Z request and response lifecycle directly to your local terminal, fully supporting asyncio.

1. Enabling Dev Mode

To start mocking, simply set dev_mode=True in your global client. You do not need to provide an API key. 

from syncforge import SyncForge

# api_key is ignored in dev_mode
sf = SyncForge(dev_mode=True, backend='in_memory')

2. A-Z Event Mocks & Terminal Logs

Because SyncForge intercepts every FastAPI Async Dependency, it will generate mocked JSON payloads in your console asynchronously. Here is the comprehensive breakdown of every feature's request and response log.

Feature: Async Table Registration

sf.create_table()

Triggered when your FastAPI startup event registers a new table structure.

🌐 [SyncForge Local Dev] Simulated Network Request
► METHOD: POST
► URL:    https://syncforge.dev/api/v1/sync/products/
◄ RESPONSE: {
  "success": true,
  "status": "ok",
  "table": "products",
  "message": "[Dev Mode] Table Created Async",
  "data": {
    "action": "create",
    "version": 1
  }
}

Feature: Background Coalescing Sync

Depends(get_sf)

When a FastAPI dependency triggers a background cache sync update.

🌐 [SyncForge Local Dev] Simulated Network Request
► METHOD: POST
► URL:    https://syncforge.dev/api/v1/sync/orders/
◄ RESPONSE: {
  "success": true,
  "status": "ok",
  "table": "orders",
  "message": "[Dev Mode] Async Refresh Completed",
  "data": {
    "action": "refresh",
    "coalesced_threads": 4
  }
}

Switching back to Live (Cloud) Mode

When you are ready to deploy or want to see live analytics in the SyncForge Dashboard, you must switch back to Live Mode.

  1. Remove dev_mode=True from your initialization.
  2. Provide your valid SYNCFORGE_API_KEY (starting with sf_live_).
  3. Once switched, terminal prints will stop, and all logs will instantly begin streaming to your Remote Dashboard CMD.
# Back to Live Cloud Mode
sf = SyncForge(
    api_key=os.environ.get('SYNCFORGE_API_KEY'),
    backend='redis'
)
