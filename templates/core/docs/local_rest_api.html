REST API Local (Offline) Mode

If you are integrating SyncForge via direct HTTP calls (curl, axios, Go, Rust), you can still simulate the REST environment locally. By running the SyncForge Local Engine image, all network calls to localhost:8000 are handled offline without touching the cloud.

1. A-Z Event Mocks & Terminal Logs

When hitting the local emulator, you receive identical JSON shapes as you would in production, ensuring your code works seamlessly when deployed.

Feature: Trigger Sync via cURL

POST /api/v1/sync/

Sending an offline request to invalidate a table.

$ curl -X POST http://localhost:8000/api/v1/sync/inventory/

◄ RESPONSE: {
  "success": true,

Switching back to Live (Cloud) Mode

When you are ready to deploy or want to see live analytics in the SyncForge Dashboard, you must switch back to Live Mode.

  1. Change your base URL from http://localhost:8000 to https://syncforge.dev.
  2. Add the Authorization: Bearer sf_live_... header to all requests.
  3. Once switched, emulator prints will stop, and all logs will instantly begin streaming to your Remote Dashboard CMD.
# Back to Live Cloud Mode
curl -X POST https://syncforge.dev/api/v1/sync/inventory/ \
  -H "Authorization: Bearer sf_live_abc123..."
