REST API Local (Offline) Mode

If you are integrating SyncForge via direct HTTP calls (curl, axios, Go, Rust), you can still simulate the REST environment locally. By running the SyncForge Local Engine image, all network calls to localhost:8000 are handled offline without touching the cloud.

1. A-Z Event Mocks & API Responses

When hitting the local emulator, you receive identical JSON shapes as you would in production, ensuring your code works seamlessly when deployed.

Feature: Trigger Sync via cURL

POST /api/v1/sync/

Sending an offline request to invalidate a table.

bash 
$ curl -X POST http://localhost:8000/api/v1/sync/inventory/
Response 
RETURNS: {
  "success": true,
  "status": "ok",
  "table": "inventory",
  "message": "[Dev Mode Emulator] Sync Triggered",
  "data": {
    "action": "refresh"
  }
}

Feature: Register Table

POST /api/v1/tables/

When the application registers a new table structure locally.

bash 
$ curl -X POST http://localhost:8000/api/v1/tables/ \
  -H "Content-Type: application/json" \
  -d '{"table_name": "example_table"}'
Response 
RETURNS: {
  "success": true,
  "created": true,
  "status": "ok",
  "message": "[Dev Mode] Table Created Locally",
  "data": {
    "table_name": "example_table"
  }
}

Feature: Project Info

Feature: Fetch All Tables

GET /api/v1/tables/all/

Returns a list of all tables registered in your current environment (Live or Local).

bash 
$ curl -X GET http://localhost:8000/api/v1/tables/all/
Response 
RETURNS: [
  "core_product",
  "auth_user",
  "inventory"
]

Feature: Check Table Existence

POST /api/v1/tables/filter/

Quickly verifies the existence of tables, returning a smart Dict[str, bool] mapping.

bash 
$ curl -X POST http://localhost:8000/api/v1/tables/filter/ \
  -H "Content-Type: application/json" \
  -d '{"names": ["core_product", "fake_table"]}'
Response 
RETURNS: {
  "core_product": true,
  "fake_table": false
}

Feature: Clear Local Cache Tracker

DELETE /api/v1/tables/clear/

Wipes the local cache and table registry, cleanly resetting your local state.

bash 
$ curl -X DELETE http://localhost:8000/api/v1/tables/clear/
Response 
RETURNS: {
  "success": true,
  "status": "ok",
  "message": "[Dev Mode] Local tables cleared."
}

Switching back to Live (Cloud) Mode

When you are ready to deploy or want to see live analytics in the SyncForge Dashboard, you must switch back to Live Mode.

  1. Change your base URL from http://localhost:8000 to https://syncforge.dev.
  2. Add the Authorization: Bearer sf_live_... header to all requests.
  3. Once switched, emulator prints will stop, and all logs will instantly begin streaming to your Remote Dashboard CMD.
# Back to Live Cloud Mode
curl -X POST https://syncforge.dev/api/v1/sync/inventory/ \
  -H "Authorization: Bearer sf_live_abc123..."
