API Reference

Python SDK Reference

Comprehensive reference for all methods available on the SyncForge client object.

Self-Contained & Framework-Agnostic

SyncForge requires zero external infrastructure (no Redis, Kafka, or RabbitMQ required). It uses a static backend selection at initialization ('in_memory', 'django_cache', 'redis') and behaves identically across all Python frameworks. Redis is purely optional.

Initialization

SyncForge(**kwargs)

Initializes the main engine client.

Parameter Type Description
api_keystr (required)Your SyncForge project API key starting with `sf_live_`.
backendstrDefault `'memory'`. Supports `'django_cache'` or `'redis'`.
silentboolDefault `False`. If `True`, network errors are caught and logged as warnings.
async_modeboolDefault `False`. If `True`, refresh calls run in fire-and-forget background threads.
encryption_keystrOptional 32-byte secret. Enables military-grade AES-256 encryption on all local disk cache files.
sign_requestsboolDefault `True`. Protects outgoing API requests with HMAC replay attack signatures.

Reading Data

sf.cache_query()

The heart of SyncForge's read path. A cache-aside implementation with built-in stampede protection. On a cache miss, it executes the DB query and automatically saves the result.

Parameter Type Description
table_namestrThe database table name.
querysetIterableThe Django QuerySet or SQLAlchemy query to execute on cache miss. The cache key is securely auto-generated using a SHA-256 hash of this query.
timeoutint (default 3600)Time in seconds before the RAM cache expires.
Example 
data = sf.cache_query(
    table_name='products',
    queryset=Product.objects.filter(active=True),
    timeout=3600
)

Output: Returns a list of model instances.

sf.get_table()

Instantly fetches pre-warmed data from local memory or disk without triggering a database query. 

data = sf.get_table('products')

Output: Returns the cached list of model instances, or None if not found.

Invalidating Data

sf.refresh()

Signal SyncForge that data in one or more tables has changed. Call this after any database write. This function can be called from inside your web application OR from any standalone Python script outside the web application. 

# Works inside Django/Flask/FastAPI or in standalone scripts:
from syncforge import SyncForge
sf = SyncForge(api_key="sf_live_...", backend="in_memory")

# Invalidate cache for tables:
result = sf.refresh('products', 'categories')

Output: Returns a SyncResult object (or list of them) containing metrics like calls_saved.

Anti-Spam Cooldown Rate Limiting: Refreshes are restricted to at most 1 request per 60 seconds (1 minute) per table. If you exceed this rate limit (via script or by spamming the manual refresh button in the dashboard), the API returns a 429 Too Many Requests response, raising a RateLimitError.
Enterprise Feature — Request Coalescing: SyncForge automatically coalesces concurrent identical sf.refresh() calls made within a 2-second window. A massive stampede of 50 background workers triggering invalidation simultaneously will result in exactly one network request, automatically shielding your API layer.

Automatic Maintenance & Middleware

4:00 AM Nightly Cache Refresh

SyncForge natively runs a Python daemon in the background of your application. Every night precisely at 4:00 AM, this daemon automatically purges all stale RAM memory and executes a fresh bulk-preload from your persistent disk cache. This guarantees your RAM doesn't become permanently bloated over the lifespan of your app, and ensures your cache is perfectly hot and ready for daily morning and afternoon traffic spikes without causing sudden latency drops.

Framework Middleware (The First Request Preload)

While SyncForge background-preloads encrypted disk caches into RAM automatically, you can explicitly guarantee that the cache is 100% hot before the very first user is served by using our Middleware.

How it works: The middleware intercepts the very first HTTP request that hits your server. Before routing the request to your views, it invokes the SyncForge C-optimized AES decrypter to immediately pump the on-disk cache into RAM. Once loaded, the request continues normally, ensuring absolute zero-latency for the user.

Django

# settings.py
MIDDLEWARE = [
    # ...
    'syncforge.django.SyncForgePreloadMiddleware',
]

FastAPI

# main.py
from fastapi import FastAPI
from syncforge.fastapi import SyncForgePreloadMiddleware
from sf_client import sf

app = FastAPI()
app.add_middleware(SyncForgePreloadMiddleware, sf_client=sf)

Flask

Flask handles preloading automatically when you initialize the SyncForgeFlask extension.

# app.py
from flask import Flask
from syncforge.flask import SyncForgeFlask
from sf_client import sf

app = Flask(__name__)
sf_ext = SyncForgeFlask(app, sf) # Triggers preloading on first request

Telemetry & Analytics

Non-Intrusive Telemetry Piggybacking

SyncForge Enterprise automatically tracks your Cache Hit Ratios, Bandwidth Saved, and Duplicates Prevented. To ensure zero performance penalty, these analytics are never sent as standalone API requests. Instead, they are piggybacked as HTTP headers (`X-SF-Telemetry`) onto existing background synchronization signals.

This means your application receives advanced observability absolutely free of I/O latency or networking overhead.

Management & Health

sf.ping()

Checks if the SyncForge API is reachable and your API key is valid.

Returns: bool (True if successful)

sf.project_info()

Fetches project metadata and all registered tables for your API key.

Returns: dict

sf.list_tables()

Returns a list of all tables registered in your project.

Returns: list[dict]

sf.speed(queryset=None, registered_table=None, disk=False)

Runs a real-time, zero-knowledge speed benchmark comparing your Database fetch time vs the SyncForge Cache fetch time. Prints the comparison beautifully to your local terminal. The dummy cache file is automatically and instantly deleted after the benchmark runs.

1. Full DB vs RAM Benchmark (Live Query)

# Runs the query, saves to dummy file, calculates both speeds
sf.speed(User.objects.all())

2. Cache-Only Benchmark (Local Read Speed)

# Skips the DB completely and only tests how fast your local cache is
sf.speed(registered_table="users")

3. Disk I/O Benchmark (Physical File Speed)

# Bypasses RAM to test physical disk encryption & decryption speed
sf.speed(User.objects.all(), disk=True)

Returns: None (Outputs directly to standard out/terminal)

sf.delete_table(name)

Permanently removes a table from your SyncForge dashboard.

Returns: bool (True if deleted)
