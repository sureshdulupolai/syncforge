Core Feature

cache_query()

The heart of SyncForge's read path. A cache-aside implementation with built-in stampede protection.

1. How it works

  1. 1
    Check Django cache for cache_key.
  2. 2
    If cache hit: return data instantly. Fire background thread to report cache hit to SyncForge API.
  3. 3
    If cache miss: acquire lock (stampede protection), execute database queryset.
  4. 4
    Store result in cache for timeout seconds.
  5. 5
    Register cache_key under table_name registry so sf.refresh() knows what to clear later.

2. Basic usage (Explicit If/Else)

It is best practice to explicitly check if the data exists in the cache using an if/else block. This guarantees that the database is only queried for the very first client. Every subsequent client will receive the exact same formatted data directly from the fast cache, saving massive database load.

views.py 
from django.core.cache import cache
from myproject.sf import sf
from .models import Product

def product_list(request):
    cache_key = 'all_active_products'
    
    # 1. First, check if data is already in cache
    data = cache.get(cache_key)
    
    # 2. If/Else check: If data is NOT in cache (First Client Only)
    if data is None:
        # Fetch from Database and apply filters/ordering
        data = list(Product.objects.filter(active=True).order_by('name'))
        
        # Save EXACTLY in this format to cache for 1 hour
        cache.set(cache_key, data, 3600)
        
        # Tell SyncForge to track this key so it can be auto-cleared on updates
        sf.track_key(table_name='core_product', cache_key=cache_key, timeout=3600)
    
    # 3. Return data (Subsequent clients skip the DB entirely)
    return render(request, 'list.html', {
        'products': data,
        'count': len(data)
    })

3. Dynamic / parameterized keys

If your data depends on user input (e.g. pagination or filters), the cache_key must include those parameters to avoid mixing up different data formats.

views.py 
def category_list(request, category_id):
    # BAD: All categories will share the same cache key!
    # key = 'category_items' 

    # GOOD: Key includes the parameter (ensures separate cache for each category)
    key = f'category_items_{category_id}'

    data = cache.get(key)
    if data is None:
        data = list(Product.objects.filter(category_id=category_id))
        cache.set(key, data, 3600)
        sf.track_key(table_name='core_product', cache_key=key)
        
    return render(request, 'list.html', {'products': data})

4. Invalidating dynamic keys

You do not need to manually invalidate category_items_1, category_items_2, etc. When you call sf.refresh('core_product'), SyncForge clears all cache keys registered under that table name automatically.

5. Cache Stampede Protection

The Problem:

If a popular cache key expires or is refreshed, and 500 users request that page at the exact same millisecond, all 500 requests will experience a cache miss and hit your database simultaneously, bringing it down.

The Solution (built-in):

SyncForge uses Python threading.Lock() per cache key with Double-Checked Locking. Only the first thread executes the database query. The other 499 threads wait for the lock, then instantly retrieve the result from the cache populated by the first thread. Result: exactly 1 database call.

6. Simplified Auto-Key Fetching

If your table has a single global cache (no dynamic parameters like user_id), you can omit the cache_key entirely. SyncForge will auto-generate a key, and you can fetch the data later using sf.get_table() without remembering the key string.

views.py 
def get_global_report(request):
    # 1. Fetch using the auto-generated key
    data = sf.get_table("core_report")
    
    # 2. If not found, run the expensive query and cache it automatically
    if not data:
        data = sf.cache_query(
            table_name='core_report',
            queryset=Report.objects.all(),
            # cache_key omitted! Auto-generates as 'sf_auto_core_report'
        )
        
    return render(request, 'report.html', {'report': data})
