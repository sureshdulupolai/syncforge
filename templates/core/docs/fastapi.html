Framework Guide

FastAPI Integration

Use SyncForge with async FastAPI routes. The SDK's refresh() call is synchronous but runs in a background thread via async_mode=True — safe inside async endpoints.

1. Install

terminal 
pip install syncforge fastapi uvicorn sqlalchemy

2. Initialise the client

app/sf.py 
import os
from syncforge import SyncForge

# async_mode=True: refresh() launches a daemon thread and returns immediately.
# Recommended for FastAPI (no blocking inside async routes).
sf = SyncForge(
    api_key=os.environ['SYNCFORGE_API_KEY'],
    silent=True,
    async_mode=True,   # fire-and-forget — safe inside async endpoints
)

3. App lifespan & startup

main.py 
from contextlib import asynccontextmanager
from fastapi import FastAPI
from app.sf import sf
import logging

logger = logging.getLogger(__name__)

@asynccontextmanager
async def lifespan(app: FastAPI):
    if sf.ping():
        logger.info('SyncForge connected.')
    else:
        logger.warning('SyncForge unreachable — running in degraded mode.')
    yield

app = FastAPI(title='My API', lifespan=lifespan)

4. Routes — full CRUD with refresh

app/routers/products.py 
from fastapi import APIRouter, Depends, HTTPException
from pydantic import BaseModel
from app.sf import sf

router = APIRouter(prefix='/products', tags=['Products'])

# CREATE
@router.post('/', status_code=201)
async def create_product(data: ProductCreate, db=Depends(get_db)):
    product = Product(**data.model_dump())
    db.add(product); db.commit(); db.refresh(product)
    sf.refresh('products')   # async_mode=True → daemon thread, returns immediately
    return product

# UPDATE
@router.put('/{product_id}')
async def update_product(product_id: int, data: ProductCreate, db=Depends(get_db)):
    product = db.query(Product).filter(Product.id == product_id).first()
    if not product: raise HTTPException(404)
    [setattr(product, k, v) for k, v in data.model_dump().items()]
    db.commit()
    sf.refresh('products')
    return product

# DELETE
@router.delete('/{product_id}', status_code=204)
async def delete_product(product_id: int, db=Depends(get_db)):
    product = db.query(Product).filter(Product.id == product_id).first()
    db.delete(product); db.commit()
    sf.refresh('products')
Important: cache_query() uses Django's cache framework internally and is only available in Django projects. In FastAPI, use your own cache layer (e.g. aiocache, redis-py) and call sf.refresh() after every write to signal data changes to the SyncForge server.
