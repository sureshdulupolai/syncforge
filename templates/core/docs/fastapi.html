Framework Guide

FastAPI Integration

Integrate SyncForge into your FastAPI application using Dependency Injection and WAF middleware.

1. Installation

pip install syncforge fastapi uvicorn

2. Setup Client and Middleware

Initialize the SyncForge client and attach the WAF middleware to protect your endpoints.

main.py 
from fastapi import FastAPI, Depends
from syncforge import SyncForge
from syncforge.fastapi import SyncForgeWAFMiddleware, SyncForgeMaintenanceMiddleware, get_syncforge
import os

# Initialize SyncForge once globally
sf = SyncForge(
    api_key=os.environ['SYNCFORGE_API_KEY'], 
    backend='in_memory',  # Fully self-contained cache without Redis
    async_mode=True
)

app = FastAPI()
app.add_middleware(SyncForgeWAFMiddleware, sf_client=sf)
app.add_middleware(SyncForgeMaintenanceMiddleware, sf_client=sf) # Ultra-fast 4 AM IST Auto Cleanup

Client Configuration Parameters

api_key Required (str)

Your project's SyncForge API key starting with sf_live_.

timeout Default: 10 (int)

Network timeout in seconds for reaching the central SyncForge server.

silent Default: False (bool)

If True, exceptions are caught, logged, and suppressed to prevent crashing your production app.

async_mode Default: False (bool)

If True, all `refresh()` operations run in background daemon threads (fire-and-forget).

sign_requests Default: True (bool)

Uses HMAC signatures to protect outgoing server signals against replay attacks.

encryption_key Optional (str)

A secret 32-byte key used to AES-256 encrypt local cache files at rest.

cache_dir Optional (str)

Overrides the default hidden OS-specific storage path where encrypted cache binaries are safely written.

Connection Backends backend_type="..."

  • 'in_memory': Recommended for single-node ASGI. Ultra-fast RAM dictionary. Lowest latency.
  • 'redis': Connects directly to a Redis cluster via the redis_url argument. Recommended for multi-worker Uvicorn clusters.

3. Using get_table() in routes

Use FastAPI's Depends to inject the client and use the cache-aside pattern.

routers/products.py 
@app.get("/products")
async def list_products(sf_client: SyncForge = Depends(get_syncforge(sf))):
    # 1. Fetch from RAM Cache instantly
    data = sf_client.get_table("users")
    
    if not data:
        # 2. Cache Miss: Execute DB query and automatically save to memory
        data = sf_client.cache_query(
            table_name="users",
            queryset=await db.fetch_all("SELECT * FROM users"),
            timeout=3600
        )
    return {"users": data}
    
@app.post("/users")
async def create_user(payload: ProductSchema, sf_client: SyncForge = Depends(get_syncforge(sf))):
    # ... DB Write ...
    await db.execute("INSERT INTO users...")

    # 3. Refresh cache across cluster instantly
    sf_client.refresh("users")
    return {"status": "created"}

Enterprise Table Parameters Explained

When registering a table via sf.create_table(), you can use these enterprise features:

sync_mode

What it does: Determines when the SyncForge server is notified. Values: 'event' (on every DB write), 'manual' (only via sf.refresh()), 'schedule_1h', etc.

storage_mode

What it does: 'ram_disk' (default) automatically preloads data from the encrypted disk cache into RAM via background threads, providing zero-latency reads while backing up persistently.

encryption / compression

What it does: Applies AES-256 encryption at rest (True by default). Use compression='lz4' for massive querysets to dramatically reduce I/O wait times.
