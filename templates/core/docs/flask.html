Framework Guide

Flask Integration

Use SyncForge with Flask routes, Blueprints, and SQLAlchemy. No special Flask extension required — just the core SDK.

1. Install

terminal 
pip install syncforge flask sqlalchemy

2. Initialise the client

app/sf.py 
import os
from syncforge import SyncForge

sf = SyncForge(
    api_key=os.environ['SYNCFORGE_API_KEY'],
    silent=True,
)

3. Application factory

app/__init__.py 
from flask import Flask
from .sf import sf

def create_app():
    app = Flask(__name__)

    # Verify connectivity on startup
    if not sf.ping():
        app.logger.warning('SyncForge unreachable — running without sync')

    from .routes import products_bp
    app.register_blueprint(products_bp)

    return app

4. Reading Data (Cache-Aside Pattern)

SyncForge uses a smart cache-aside pattern to protect your database. Follow these 3 steps in your Flask routes:

  1. 1
    get_table(): Instantly fetches pre-warmed data from local memory. Zero database hits.
  2. 2
    cache_query(): If memory is empty (cache miss), executes the DB query and automatically saves the result back to memory.
  3. 3
    Combine Both: Use a simple if/else block to ensure only the very first user triggers a database query.
app/routes/products.py 
from flask import Blueprint, request, jsonify
from app.sf import sf
from app.models import Product, db

products_bp = Blueprint('products', __name__, url_prefix='/products')

# READ (Cached)
@products_bp.get('/')
def list_products():
    if sf.get_table('products'):
        # 1. Cache Hit: Data found in memory! DB query skipped entirely.
        products = sf.get_table('products')
    else:
        # 2. Cache Miss: Execute DB query and automatically save to memory
        products = sf.cache_query(
            table_name='products',
            queryset=Product.query.all()
        )
        
    # 3. Return data exactly as it came from the DB
    return jsonify([p.to_dict() for p in products])

# CREATE
@products_bp.post('/')
def create_product():
    data    = request.get_json()
    product = Product(name=data['name'], price=data['price'])
    db.session.add(product)
    db.session.commit()
    sf.refresh('products')    # ← clears local cache and notifies server
    return jsonify(product.to_dict()), 201

# UPDATE
@products_bp.put('/')
def update_product(pk):
    product       = Product.query.get_or_404(pk)
    product.price = request.get_json().get('price', product.price)
    db.session.commit()
    sf.refresh('products')
    return jsonify(product.to_dict())

# DELETE
@products_bp.delete('/')
def delete_product(pk):
    product = Product.query.get_or_404(pk)
    db.session.delete(product)
    db.session.commit()
    sf.refresh('products')
    return '', 204

5. Refresh multiple tables at once

python 
# Single call refreshes all three tables
sf.refresh('products', 'categories', 'inventory')
