Framework Guide

Flask Integration

Integrate SyncForge into your Flask application using the SyncForgeFlask extension. This automatically handles WAF security middleware, request tracking, and the new ultra-fast 4 AM IST Maintenance cache cleanup.

1. Install

pip install syncforge flask

2. Initialise the extension

app.py 
from flask import Flask, jsonify
from syncforge import SyncForge
from syncforge.flask import SyncForgeFlask
import os

# Initialize the SyncForge Engine
sf = SyncForge(
    api_key=os.environ.get("SYNCFORGE_API_KEY"),
    backend='in_memory',  # Zero external dependencies required
    async_mode=False
)

app = Flask(__name__)
sf_ext = SyncForgeFlask(app, sf)

Client Configuration Parameters

api_key Required (str)

Your project's SyncForge API key starting with sf_live_.

timeout Default: 10 (int)

Network timeout in seconds for reaching the central SyncForge server.

silent Default: False (bool)

If True, exceptions are caught, logged, and suppressed to prevent crashing your production app.

async_mode Default: False (bool)

If True, all `refresh()` operations run in background daemon threads (fire-and-forget).

sign_requests Default: True (bool)

Uses HMAC signatures to protect outgoing server signals against replay attacks.

encryption_key Optional (str)

A secret 32-byte key used to AES-256 encrypt local cache files at rest.

cache_dir Optional (str)

Overrides the default hidden OS-specific storage path where encrypted cache binaries are safely written.

Connection Backends backend_type="..."

  • 'in_memory': Recommended for single-node Flask. Ultra-fast RAM dictionary. Lowest latency. Cleared on server restart.
  • 'redis': Connects directly to a Redis cluster via the redis_url argument.

3. Cache-Aside Pattern with get_table()

Always use the 3-step Cache-Aside pattern in your routes to ensure optimal performance. 1. Check if the table exists in memory cache using sf.get_table(). 2. If it does not exist, use sf.cache_query() to hit the DB and store the result.

routes.py 
@app.route("/products", methods=["GET"])
def list_products():
    # 1. Cache Hit: Data found in memory! DB query skipped entirely.
    products = sf.get_table("products")
    
    if not products:
        # 2. Cache Miss: Execute DB query and automatically save to memory
        products = sf.cache_query(
            table_name="products",
            queryset=db.session.query(Product).all(),
            timeout=3600
        )
        
    return jsonify({"products": products})
    
@app.route("/products", methods=["POST"])
def create_product():
    # ... Write to Database ...
    # 3. Refresh cache across cluster instantly
    sf.refresh("products")
    return jsonify({"status": "created"})

Enterprise Table Parameters Explained

When registering a table via sf.create_table(), you can use these enterprise features:

sync_mode

What it does: Determines when the SyncForge server is notified. Values: 'event' (on every DB write), 'manual' (only via sf.refresh()), 'schedule_1h', etc.

storage_mode

What it does: 'ram_disk' (default) automatically preloads data from the encrypted disk cache into RAM via background threads, providing zero-latency reads while backing up persistently.

encryption / compression

What it does: Applies AES-256 encryption at rest (True by default). Use compression='lz4' for massive querysets to dramatically reduce I/O wait times.

Switching to Local (Offline) Mode

Developing against the Live Cloud ensures you see real dashboard analytics. However, if you are flying, working completely offline, or want to avoid spamming your live project with test data, you can seamlessly switch your Flask project to Local Dev Mode.

syncforge_client.py 
# Instantly drop all network calls. No API key required.
sf = SyncForge(dev_mode=True, backend='in_memory')

What happens when you switch?

  • SyncForge instantly stops routing requests to the cloud edge.
  • All Flask extensions and routes continue behaving identically.
  • Terminal requests and responses are perfectly mocked and logged directly to your standard output.
