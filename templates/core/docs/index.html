Introduction

SyncForge Documentation

SyncForge is a developer-controlled data synchronisation platform. Instead of expiring data on a timer, you decide exactly when your data becomes stale — by calling sf.refresh().

How it works — in one diagram

First request
──────────────
cache_query() → cache MISS → DB query → store in cache → return data

All subsequent requests (until refresh)
────────────────────────────────────────
cache_query() → cache HIT → return instantly   ← No DB query

After sf.refresh("products")
──────────────────────────────
cache_query() → cache MISS → DB query → store fresh data → return
